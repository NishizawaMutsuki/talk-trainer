# Talk Trainer ナレッジ抽出 — SKILL.md
# ═══════════════════════════════════════════════════════════════
#
# ■ 目的
# 面接動画から「学習者が自分で本質を掴める具体」を抽出・蓄積する。
# このファイル自体が抽出作業の品質基準であり、新しい動画を処理する際に
# 必ず参照するガイドラインである。
#
# ═══════════════════════════════════════════════════════════════
# ■ 最重要原則: 抽象化しない
# ═══════════════════════════════════════════════════════════════
#
# ✗ やってはいけないこと:
#   「行動力が大事」「具体的に話しましょう」「数字を使おう」
#   → 就活本に書いてある。誰でも言える。学習者の行動を変えない。
#
# ✓ やるべきこと:
#   元Amazon人事が「何百人もESを見た後に名前を覚えている学生は、
#   オリーブオイルに目覚めて四国の農家と契約した子だった」と語った。
#   → 学習者が「あ、つまり規模じゃなくて"なぜそこまで?"という
#     突き抜け方が記憶に残るのか」と自分で気づける。
#
# 抽出者の仕事は「要約」ではなく「選別」である。
# 本質を突いた具体を見つけ、そのまま残す。
# 学習者の抽象化の権利を奪わない。
#
# ═══════════════════════════════════════════════════════════════
# ■ 抽出基準: 何を残し、何を捨てるか
# ═══════════════════════════════════════════════════════════════
#
# 残すもの:
#   1. 人事・面接官の「本音」— 建前ではなく内側の判断基準が見えるもの
#      例: 「ノートに"多分AIに聞いてたと思う"って書く」
#   2. 具体的なエピソード — 抽象的な教訓ではなく、場面が浮かぶもの
#      例: バイト先で売れ残りの服の行方が気になりバングラデシュの工場まで行った学生
#   3. 受かる/落ちる の分岐点 — 同じ状況で結果が分かれた具体的な差
#      例: 「インプレッション何%アップ」と書いても、その業界じゃない人事には通じない。
#           「一般的にはこうで、それをこうしました」と補足した人は印象に残った。
#   4. 面接官の感情が動いた瞬間の描写
#      例: 「面接官のLinkedInを事前に見て"海外の大学出てたんですね"と聞かれると
#            しめしめって思う」
#   5. 逆説・意外性 — 常識と異なる観察
#      例: 「失敗した話をそのまま話せる人の方がよっぽどすごいと思う」
#
# 捨てるもの:
#   - 動画の導入・宣伝・雑談・チャンネル登録促進
#   - 「〜が大事です」で終わる抽象的アドバイス（具体が伴わないもの）
#   - 既に広く知られている一般的な就活マナー（敬語、時間厳守 等）
#   - 他の動画と重複する内容（初出のみ残す）
#
# ═══════════════════════════════════════════════════════════════
# ■ エントリ形式
# ═══════════════════════════════════════════════════════════════
#
# 各エントリはJSONL形式で knowledge/insights.jsonl に追記する。
#
# {
#   "id": "連番",
#   "source_id": "YouTubeビデオID",
#   "source_title": "動画タイトル",
#   "speaker_role": "発言者の立場（元Amazon人事 / リクルート元人事責任者 / 内定者 等）",
#   "context": "どういう話の流れで出てきたか（1-2文）",
#   "insight": "核心をついた具体的な発言・エピソード（原文に忠実に。ただし字幕ノイズは除去）",
#   "tags": ["検索用タグ"],
#   "tier": "重要度 S/A/B（S=面接の本質に関わる / A=実践的で有用 / B=参考になる）"
# }
#
# ═══════════════════════════════════════════════════════════════
# ■ 品質チェックリスト（1本の動画を処理した後に確認）
# ═══════════════════════════════════════════════════════════════
#
# □ 各insightを読んで、具体的な場面・人物・行動が浮かぶか？
#   → 浮かばないなら抽象化しすぎ。原文に戻って具体を拾い直す。
#
# □ 「〜が大事」「〜すべき」で終わっているinsightはないか？
#   → あるなら「なぜそう言えるのか」の根拠となる具体に差し替える。
#
# □ 学習者がこのinsightを読んで「自分の回答をこう変えよう」と
#   具体的にイメージできるか？
#   → できないなら、insightの粒度が大きすぎる。分割するか具体を足す。
#
# □ このinsightは他の就活本やサイトにも書いてある一般論か？
#   → そうなら削除。この動画ならではの具体でなければ価値がない。
#
# ═══════════════════════════════════════════════════════════════
# ■ 運用ルール
# ═══════════════════════════════════════════════════════════════
#
# 1. 新しい動画を処理する前に、必ずこのSKILL.mdを読み返す
# 2. 1本の動画から抽出するinsightは5〜15個程度（多すぎると質が落ちる）
# 3. 抽出後、既存のinsights.jsonlと重複がないか確認する
# 4. 重複がある場合、より具体的・鮮明な方を残す
# 5. このSKILL.md自体も、作業を通じて改善していく
#
