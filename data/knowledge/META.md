# Talk Trainer 知識ベース — 運用ガイド（META）

## ファイル構成

```
data/knowledge/
├── META.md           ← このファイル（運用ルール）
├── COACHING.md       ← メイン知識ベース（面接の具体的知見）
├── PATTERNS.md       ← （将来）回答パターン集
└── COMPANY_RESEARCH.md ← （将来）企業別求める人材像
```

## 設計思想

### 核心原則：「抽象化しない」
- ❌ 「具体的に話しましょう」
- ✅ 「2週間に1回ですと答えて面接官に物足りなく思われた。本当は頻度ではなくのめり込み度を聞いている」

### 何を保存するか
1. **場面（Scene）**: いつ・誰が・何の質問に対して
2. **発言（Quote）**: 面接官・学生が実際に言った言葉
3. **判断（Judgment）**: 面接官がなぜ丸/バツをつけたか、その具体的根拠
4. **対比（Contrast）**: 同じ質問に対する「落ちる回答」と「受かる回答」の具体差分

### 何を保存しないか
- 「結論から話しましょう」のような汎用アドバイス
- 出典のない一般論
- 面接テンプレート（構造は具体例から学習者が抽出する）

## 追加手順

### 新しい動画から知見を追加する場合

1. `subs_clean/` のテキストを精読
2. 以下の「抽出チェックリスト」に沿って具体を拾う
3. COACHING.md の該当セクションに連番で追記
4. メタデータテーブルの処理済みに追加

### 抽出チェックリスト
各動画を読む際、以下の具体を探す：

- [ ] 面接官が「この人覚えた」と言った瞬間とその理由
- [ ] 面接官が「ここで落とす判断をした」瞬間とその理由
- [ ] 学生の回答に対する面接官のリアルタイム反応（感嘆、困惑、疑問）
- [ ] 「Before → After」の回答改善例
- [ ] 面接官が「実はこれを見ている」と明かした隠れた評価軸
- [ ] 質問の「表面的な意味」と「本当に知りたいこと」のギャップ
- [ ] 特定の業界・企業での固有の評価基準
- [ ] 合格者と不合格者の具体的な差分

### セクション構成ルール
- セクション番号: `## N. セクション名`
- エントリ番号: `### N-M. エントリタイトル`
- 各エントリに必須: **出典**（動画ID）、**場面**、**核心**
- 推奨: **面接官の発言**（引用）、**対比**（良い例と悪い例）
